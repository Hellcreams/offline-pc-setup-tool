# 사용설명서 (관리자)
## 폐쇄망 윈도우 작업 자동화
### 1. WIM 이미지 파일 만드는 법
1) 설치 대상 OS를 직접 키고, 윈도우 실행 창에 `sysprep`라고 입력하고 실행시킨다.
2) `sysprep.exe`를 실행시킨다.
3) 옵션을 다음과 같이하고, 컴퓨터가 꺼질 때까지 기다린다.
	* 시스템 정리 작업 : 시스템 감사 모드 시작
	* 일반화 : 체크
	* 종료 옵션 : 시스템 종료
4) 윈도우 PE 모드로 들어간다.
5) GimageX를 실행시키고, 옵션을 다음과 같이 설정한다.
	* 소스 경로 : 윈도우 파티션 (보통은 C:)
	* 대상 파일 : 이미지 저장 경로
	* 이미지 이름 : 식별 가능한 형태로 (ex. '폐쇄망', '회사 내부망'...)
	* 부팅가능 : 체크 X
	* 유효성 검사 : 체크
	* 무결성 확인 : 체크
6) '이미지 생성'을 눌러 이미지를 만든다.
* 주의점
	* `sysprep.exe`는 인터넷망에 한번도 연결되어 있지 않은 상태여야 실행 가능하다.
	* `sysprep.exe` 실행 후, PE로 들어가지 않고 원본 OS로 부팅하게 되면, 처음부터 과정을 다시 시작해야 한다. 
### 2.  이미지 적용
#### 실행법
1)  bat파일을 킨다.
2) 포맷할 디스크 번호를 확인하고 입력한다.
3) 윈도우 파티션 (C 드라이브)의 용량을 지정한다. (0을 입력하면 하드 전체 및 백업 파티션 (D 드라이브)
4) 경고창이 누르면, 포맷 대상을 다시 확인하고 진행한다.
5) 실행되는 WinNTSetup 프로그램에서, 적용할 이미지를 찾는다.
6) 재부팅하면 끝.

#### 실행법 (간편 ver.)
1) bat mini를 켠다.
2) 설치 대상 디스크 번호를 확인하고 입력한다.
3) 시스템 파티션 번호를 확인하고 지정한다.
4) 위 [실행법] 5~6번과 동일.

#### 주의점
* C, D, S가 다른 디스크에 지정되어 있다면, 할당에 오류가 생길 수 있음.
	* C가 지정된 디스크 외에 있다면 -> 디스크 관리자에서 C 할당을 조정할 것
	* D가 지정된 디스크 외에 있다면 -> 프로그램 실행에서는 무시해도 정상적으로 실행됨.
	* S가 지정된 디스크 외에 있다면 -> `reset_S.bat` 을 실행시키면 할당 해제됨
* '심각' 경고가 뜬다면 : 초기화하는 디스크가 진짜 윈도우 전용 디스크인지 확인.

#### 추가기능
* 할당된 IP가 까먹는 것을 방지하기 위해, 포맷 전 lastest_IP.txt 에 아이피 정보 파일 기록되게 만듬. 확인해봐.


### 3.  고정 아이피 세팅
#### 사용법  *(사진 추가)*
1) IP수정기에 '우클릭' -> '관리자 권한 실행' 을 누른다.
2) (해당사항 있는 경우) 해당하는 네트워크 번호를 입력한다.
3) 해당하는 숫자들을, 한칸씩 입력한다.
4) 아이피가 정확히 입력했는지 확인한다.

#### 프로그램 설정값
##### 수정법 
* 해당 bat프로그램에 '우클릭' -> '메모장에서 편집'을 누르면, 변경할 수 있다. *(사진 추가)*

##### 설정값 목록 
*(변수 목록 사진 추가)*
* SUBNET_MASK 
	*  서브넷 마스크에 해당하는 값.
* SUBNET_RANGE `[0~3]`
	* 단말기 IP주소와 게이트웨이 주소가 동일한 영역을 지정해주는 값.
	* 예를 들어 값이 3이면,  IP 주소의 앞 3칸까지 동일한 것으로 간주한다.
	```text
	Case 1.
		set SUBNET_RANGE=3
		단말기 IP : 30.10.1.2
		-> 게이트웨이 IP : 30.10.1.x
		
	Case 2.
		set SUBNET_RANGE=1
		단말기 IP : 20.1.2.3
		-> 게이트웨이 IP : 20.x.x.x
	```
* GT_IP
	* 초기 게이트웨이에 해당하는 IP값.
	* GT_IP 변수는 총 4개가 있으며, `[GT_IP1].[GT_IP2].[GT_IP3].[GT_IP4]` 꼴로 간주된다.
	```text
	Case 1.
		set GT_IP1=0
		set GT_IP2=0
		set GT_IP3=0
		set GT_IP4=6
		-> GT_IP = 0.0.0.6
	```
	* 최종 게이트웨이 주소는 [SUBNET_RANGE] 값에 따라 설정 값이 달라진다.
	```text
	Case 1.
		단말기 IP (입력받은 IP) : 30.10.1.2
		GT_IP : 0.0.0.6
		SUBNET_RANGE : 3
		-> 최종 게이트웨이 주소 : 30.10.1.x (단말기) + x.x.x.6 (GT_IP)
			-> 30.10.1.6
		
	Case 2.
		단말기 IP (입력받은 IP) : 20.1.2.3
		GT_IP : 20.0.0.1
		SUBNET_RANGE : 1
		-> 최종 게이트웨이 주소 : 20.x.x.x (단말기) + x.0.0.1 (GT_IP)
			-> 20.0.0.1
	```
* DNS1
	* 기본 설정 DNS 서버 주소값.
* DNS2
	* 보조 DNS 서버 주소값.
