# 개요
* 폐쇄망(내부망) 환경에서 반복적인 OS 설치와 네트워크 설정 작업을 자동화하기 위해 제작한 스크립트입니다.
* 이 스크립트는 다음과 같은 이유로 만들어졌습니다 : 
	* 수동 반복 설치에 의해 생기는 휴먼에러(인적오류) 방지
	* 신형 이미지 설치 프로그램(GHOST64 -> WinNTSetup) 을 활용하여, 필요 자원 감소 및 작업 시간 단축
	* 번거로운 설정을 자동화 / 간편화하여 부서 간 커뮤니케이션 비용 감소 
# 주요 실행파일 & 기능
## 1. Windows_Setup / 디스크 포맷 및 구성
### 기능
* WinNTSetup을 활용하고 주요 파라미터를 자동으로 설정합니다.
* 상황에 따라 초기화 방식이 두가지로 나뉩니다.
	* 전체 초기화 방식 (full_reset.bat) : 하드 하나를 초기화하고, 필요한 파티션 (부팅 파티션, OS 파티션, 보조 파티션)을 자동으로 분할하여 설정합니다.
	* 일부 초기화 방식 (mini_reset.bat) : OS/부팅 파티션만을 포맷하여 빠르게 설치합니다.
* 네트워크 인터페이스에 저장된 고정 IPv4 정보를 초기화 전 별도 파일에 저장합니다.

### 효과
* WinNTSetup을 활용하여, 필요 자원 감소 (GHOST64 대비, WIN10 기본 OS 기준)
	1) 이미지 용량 평균 약 35% 감소 (8GB -> 5GB)
	2) 작업 속도 약 55% 단축 (2분 40초 -> 1분 12초)
	3) 작업 중 발생하는 휴먼에러 감소
    4) 작업 후 후속 조치 건수 감소 (약 50% -> <5%) 
* 작업 중 특정 컴퓨터에게 부여된 고정 IP 유실 가능성 완화
* 필요한 정보를 미리 알려줌으로써 생기는 작업 중 실수 감소
* [순서도](OS_MIgration_Project/Windows_Setup/flowchart.svg)
## 2. IP_Setting.bat / 간소화된 IP 할당
### 기능
* 일반 사용자도 쉽게 사용할 수 있는 IP 설정 UX를 제공합니다.
* 서브넷 마스크, DNS 서버 등 내부망에서 고정값으로 활용되는 부분을 매번 입력할 필요 없도록 자동화합니다.
### 효과
* 각 부서 간, 또는 사용자와 부서 간 커뮤니케이션 비용 감소로 운영 효율화
* UX와 입력 요구사항 감소를 통한 일반 사용자 직접 설정 부담 완화
### 비고
* 이 스크립트는 특정 내부망 환경에 맞춰 제작되었습니다.

## 3. 참고사항
* 2026.4.15 ~ 2026.5.14 간 진행되었습니다.
* 실제 배포를 목적으로 진행되었으나, 아래와 같은 이유로 중단되었습니다.
	* 폐쇄망 환경 특성상 사용자 및 운영 인원과의 원활한 소통이 어려울 것으로 예상되며, 보안상의 이유로 피드백 수집에도 제한이 있었습니다.  
	* 자동화 도구 자체는 구현되었으나, 실제 배포판에서는 sysprep에 대한 동작 원리, 안정적인 배포판 설계법 등 사전 이해가 일정 수준 이상 요구되었습니다.
